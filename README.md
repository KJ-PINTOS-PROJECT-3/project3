## 💬 커밋 컨벤션
커밋 메시지는 다음과 같은 형식을 따릅니다: <type>: <내용 요약>

| 타입       | 설명                        |
|-----------|----------------------------|
| `feat`    | 새로운 기능 추가               |
| `fix`     | 버그 수정                    |
| `refactor`| 리팩토링 (기능 변화 X)         |
| `docs`    | 문서 관련 작업                |
| `test`    | 테스트 코드 추가/수정           |
| `chore`   | 설정, 빌드, 기타 잡일          |

예시 : feat: implement mmap
> 👊 **영어 한 단어 + 콜론(:) + 간결한 설명.

## 📌 GitHub Projects (칸반보드 사용)

> **모든 작업은 칸반보드 기준으로 움직입니다.**  
아래 항목 중심으로 관리됩니다:

- 🟩 `Todo`: 해야 할 일
- 🟨 `In Progress`: 진행 중
- 🟦 `In Review`: PR 리뷰 대기
- 🟪 `Test`: 테스트 중
- ✅ `Done`: 완료
- 🗂️ `Backlog`: 아직 미정인 아이디어나 작업
- 📝 `회의록`: 회의 결과 정리
- 📚 `StudyLog`: 하루 공부 내용 기록
